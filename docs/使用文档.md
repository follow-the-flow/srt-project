## 操作方式

### 手臂与机械臂关节对应

| 人体手臂关节 | Qarm 机械臂关节 | 角度范围 | 计算方式 |
| :--- | :--- | :--- | :--- |
| 手臂水平方向 | **Joint 1** (底座旋转) | 0 ~ 180° | 基于肩-肘向量在图像水平方向的分量，`90° + arctan2(dx, dy) × 0.5` |
| 肩部抬起角度 | **Joint 2** (肩部) | 0 ~ 90° | 肩部角度 = 髋部-肩部-肘部三点形成的角度，直接映射到 Joint 2 |
| 肘部弯曲角度 | **Joint 3** (肘部) | 0 ~ 90° | 肘部角度 = 180° - (肩部-肘部-手腕三点形成的角度)，反向映射（手臂伸直时肘部角度为0°） |
| 前臂旋转角度 | **Joint 4** (腕部) | 0 ~ 180° | 基于肘-腕向量与垂直方向的夹角，`90° + arctan2(dx, -dy)` |

**详细说明：**

1. **Joint 1 (底座旋转)**：控制机械臂在水平面上的朝向
   - 使用肩-肘向量在水平方向（X轴）的分量估算
   - 手臂向左伸展 → Joint 1 减小
   - 手臂向右伸展 → Joint 1 增大
   - 计算公式：`Joint 1 = clip(90 + arctan2(arm_x, arm_y) × 0.5, 0, 180)`

2. **Joint 2 (肩部)**：
   - 计算公式：`shoulder_angle = angle(髋部, 肩部, 肘部)`
   - 当手臂自然下垂时约 0°，平举时约 90°
   - 直接映射：`Joint 2 = clip(shoulder_angle, 0, 90)`

3. **Joint 3 (肘部)**：
   - 计算公式：`human_elbow = angle(肩部, 肘部, 手腕)`
   - 反向映射：`Joint 3 = clip(180 - human_elbow, 0, 90)`
   - 手臂伸直时 human_elbow ≈ 180°，Joint 3 = 0°
   - 手臂弯曲 90° 时 human_elbow ≈ 90°，Joint 3 = 90°

4. **Joint 4 (腕部)**：末端执行器旋转角度
   - 使用肘-腕向量与垂直方向的夹角估算
   - 手腕向左偏 → Joint 4 减小
   - 手腕向右偏 → Joint 4 增大
   - 计算公式：`Joint 4 = clip(90 + arctan2(forearm_x, -forearm_y), 0, 180)`

> **注意**：由于 YOLOv8-pose 只提供 2D 关键点，Joint 1 和 Joint 4 的计算是**近似估算**。要获得更精确的 3D 控制，需要使用深度摄像头或 3D 姿态估计模型。

### 姿态控制

| 你的动作 | Qarm 响应 |
| :--- | :--- |
| 抬起/放下手臂 | 机械臂肩部关节跟随 |
| 弯曲/伸直肘部 | 机械臂肘部关节跟随 |
| 移动手腕 | 机械臂腕部关节跟随 |

### 按键控制

| 按键 | 功能 |
| :--- | :--- |
| `1` | 切换到左臂控制 |
| `2` | 切换到右臂控制 |
| `+` / `=` | 放大右屏 3D 视图 |
| `-` / `_` | 缩小右屏 3D 视图 |
| `r` | 重置机械臂到初始位置 |
| `q` | 退出程序 |

2. 程序自动识别最靠近的人（最大检测框）

| 问题 | 解决方案 |
| :--- | :--- |
| 没有检测到姿态 | 站得更近/调整站位，确保上半身在画面中 |
| 识别了多个人 | 程序自动选择最靠近的（最大检测框） |
| 机械臂不动 | 确保手臂关键点（肩/肘/腕）都在画面中 |
| 画面显示不完整 | 调整站位距离，或修改 `qarm_sim.py` 中的投影参数 |

## 技术说明

### 当前版本 (v1.0)

**姿态检测：** YOLOv8-pose（2D 关键点检测）

| Qarm 关节 | 当前计算方式 | 精度 | 限制 |
| :--- | :--- | :--- | :--- |
| Joint 1 (底座) | 肩-肘向量的 2D 投影角度 | 近似 | 假设人正对摄像头，无法检测深度方向移动 |
| Joint 2 (肩部) | 髋-肩-肘三点角度 | 高 | 基于清晰的身体几何关系 |
| Joint 3 (肘部) | 肩-肘-腕三点角度（反向） | 高 | 基于清晰的身体几何关系 |
| Joint 4 (腕部) | 肘-腕向量的 2D 方向 | 近似 | 缺少手掌朝向信息 |

**优点：**
- 使用普通 RGB 摄像头即可
- 实时性能好
- 代码简洁，易于理解

**限制：**
- Joint 1 和 Joint 4 是近似值
- 无法检测前后方向（深度）的运动
- 手腕旋转信息不完整

### 未来改进方案

要获得更精确的 3D 控制，可以采用以下方案：

#### 方案 A：使用 MediaPipe Pose（推荐）

**优势：**
- 提供 33 个 3D 关键点（包含 x, y, z 坐标）
- 无需额外硬件
- 实时性能优秀

**需要修改：**
```python
# 替换 YOLOv8-pose 为 MediaPipe Pose
import mediapipe as mp

# 获取 3D 关键点
landmarks = results.pose_world_landmarks

# Joint 1: 使用肩-肘在 3D 空间中的 XY 平面投影
joint1 = calculate_horizontal_angle(shoulder_3d, elbow_3d)

# Joint 4: 使用手部检测获取更精确的腕部角度
```

**依赖安装：**
```bash
pip install mediapipe
```

#### 方案 B：使用深度摄像头

**硬件选择：**
- Intel RealSense D400 系列
- Microsoft Azure Kinect
- Orbbec Femto Bolt

**优势：**
- 真实的 3D 深度信息
- 不受光照条件影响
- 可以精确计算空间角度

**需要修改：**
```python
import pyrealsense2 as rs

# 获取深度图像
depth_frame = frames.get_depth_frame()

# 获取 3D 坐标
points = rs.pointcloud.calculate(depth_frame)

# 使用真实 3D 坐标计算关节角度
joint1 = calculate_3d_angle(shoulder_3d, elbow_3d, hip_3d)
```

#### 方案 C：添加手部检测

**目的：** 精确计算 Joint 4（腕部旋转）

**使用 MediaPipe Hands：**
```python

mp_hands = mp.solutions.hands
hands = mp_hands.Hands()

# 获取手部 21 个关键点
# 用手腕-中指指尖向量计算手掌朝向
```

#### 对比总结

| 方案 | 成本 | 精度 | 实现难度 |
| :--- | :--- | :--- | :--- |
| 当前版本 (YOLOv8) | 低 | 中 | 简单 |
| MediaPipe Pose | 低 | 高 | 中等 |
| 深度摄像头 | 高 | 很高 | 较难 |
| + 手部检测 | 低 | 中+ | 中等 |
