## YOLOv8-pose 简介

### 什么是 YOLOv8-pose

YOLOv8-pose 是 Ultralytics 公司推出的实时人体姿态检测模型，基于 YOLOv8 目标检测架构，可以同时检测多人并估计每个人的 17 个身体关键点。

### 特点

| 特点 | 说明 |
|------|------|
| **实时检测** | 单张图像推理速度快（~10ms） |
| **多人检测** | 可同时检测多人的姿态 |
| **2D 关键点** | 输出 2D 图像坐标，无深度信息 |
| **置信度评分** | 每个关键点都有置信度分数 |
| **预训练模型** | 使用 COCO 数据集训练 |

### 模型文件

| 模型 | 大小 | 速度 | 精度 |
|------|------|------|------|
| **yolov8n-pose** | 6.8MB | 最快 | 中等 |
| yolov8s-pose | 23MB | 快 | 高 |
| yolov8m-pose | 50MB | 中等 | 很高 |

**本项目使用：** `yolov8n-pose.pt` (nano 版本，平衡速度和精度)

## 输出格式

### 检测结果结构

```python
results = model(image)  # YOLO 推理
result = results[0]     # 第一个检测结果

# 关键点坐标 (归一化 0-1)
keypoints = result.keypoints.xy[person_idx]  # shape: (17, 2)
# 例如: [[0.5, 0.4], [0.5, 0.6], [0.5, 0.7], ...]

# 置信度
conf = result.keypoints.conf[person_idx]  # shape: (17,)
# 例如: [0.95, 0.92, 0.88, ...]

# 边界框
boxes = result.boxes.xyxy[person_idx]  # shape: (4,)
# 例如: [x1, y1, x2, y2]
```

### 坐标说明

| 坐标类型 | 范围 | 说明 |
|----------|------|------|
| **归一化坐标** | 0.0 ~ 1.0 | 相对于图像宽高的比例 |
| **像素坐标** | 0 ~ width/height | 实际像素位置 |
| **置信度** | 0.0 ~ 1.0 | 检测的可信程度 |

### 代码位置


### 使用流程

```python
# 1. 加载模型
from ultralytics import YOLO
model = YOLO('data/models/yolov8n-pose.pt')

# 2. 推理
results = model(image, conf=0.5, verbose=False)

# 3. 提取关键点
keypoints = results[0].keypoints.xy[0]  # 第一个人的关键点
conf = results[0].keypoints.conf[0]    # 置信度

# 4. 获取右臂关键点
right_shoulder = keypoints[6]  # (x, y) 归一化坐标
right_elbow = keypoints[8]
right_wrist = keypoints[10]

# 5. 转换为像素坐标
h, w = image.shape[:2]
shoulder_px = (int(right_shoulder[0] * w), int(right_shoulder[1] * h))
elbow_px = (int(right_elbow[0] * w), int(right_elbow[1] * h))
```

## 坐标系统

### 图像坐标系（YOLO 输出）

```
(0,0) ──────────────→ X (归一化 0-1)
  │
  │
  │
  ↓
  Y (归一化 0-1)

关键点坐标格式: (x, y)
- x: 水平位置，0=左边缘，1=右边缘
- y: 垂直位置，0=上边缘，1=下边缘
```

### 与物理坐标系的关系

| 坐标系 | X 方向 | Y 方向 | Z 方向 |
|--------|--------|--------|--------|
| **YOLO 图像** | → 向右 | ↓ 向下 | ❌ 无 |
| **物理空间** | → 向右 | ↓ 向前 | ↑ 向上 |

**转换：** 通过角度映射实现，而非直接坐标变换

## 局限性

### 1. 无深度信息

| 问题 | 影响 |
|------|------|
| 输出只有 (x, y) | 无法获取 Z 坐标（深度）|
| 透视投影误差 | 同一角度在不同深度下投影不同 |
| 肘部角度误差 | 深度变化大时误差 ±20° |

**示例：**
```
3D 空间中肘部弯曲 90°：

靠近相机 (0.5m):     远离相机 (1.5m):
    肘─腕                  肘─────腕
    │                      │
    肩                      肩

2D 投影角度: ~60°      2D 投影角度: ~85°
```

### 2. 遮挡问题

| 场景 | 检测效果 |
|------|----------|
| 正面站立 | ✅ 准确 |
| 侧面站立 | ⚠️ 关键点偏移 |
| 手臂被遮挡 | ❌ 检测失败 |
| 多人重叠 | ⚠️ 可能混淆 |

### 3. 置信度阈值

| 置信度 | 处理方式 |
|--------|----------|
| > 0.5 | ✅ 接受 |
| 0.3 ~ 0.5 | ⚠️ 边缘 |
| < 0.3 | ❌ 忽略 |

**当前设置：** `conf=0.5`（最低置信度 0.5）

## 常见问题

### Q1: 为什么只用右臂？

**A:** 简化演示，避免左右臂混淆。如需双手控制，可修改代码添加左臂支持。

### Q2: 如何提高检测精度？

**A:**
- 使用更大的模型（yolov8s-pose）
- 增加图像分辨率
- 调整置信度阈值
- 确保良好的光照条件

### Q3: 深度信息缺失如何解决？

**A:**
- 双目相机（硬件方案）
- 深度估计模型（MiDaS 等）
- 接受一定误差（当前方案）

### Q4: 检测不到人怎么办？

**A:**
- 检查相机连接
- 确认光照充足
- 调整置信度阈值
- 检查人体是否在画面中
